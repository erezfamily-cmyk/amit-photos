## 1. פאזל הזזה (puzzle/index.html)

### מה נבנה
דף משחק עצמאי (`puzzle/index.html`) — פאזל הזזה קלאסי (15-puzzle, 4×4) על תמונות של עמית. ללא תלויות חיצוניות. כל הקוד — HTML + CSS + JS בקובץ אחד.

### מסך פתיחה (Splash)
- לוגו + שם הצלם
- כפתור "התחל לשחק"
- OG tags לשיתוף (`og:title`, `og:image`, `og:description`)

### בחירת קושי
- לחצני 3×3 / 4×4 (ניתן להחליף תוך כדי משחק)
- כפתור "🔀 החלף" לתמונה אחרת

### לוח ומשחק
- קוביות ממוקמות באבסולוט עם `transition: left/top 0.12s ease` לאנימציה חלקה
- תמונות נטענות כ-`<img>` (עוקף CORB של Google Drive)
- Shimmer skeleton בזמן טעינת התמונה (אנימציה composite-only עם `will-change: transform`)
- Tile border-radius: 3px
- לחיצה על קוביה — מזיזה אם סמוכה לחלל הריק
- חצים במקלדת
- Swipe בגע (touch events) — מובייל

### שליטה מובייל
- `height: 100dvh` (תמיכה ב-iOS Safari)
- `overscroll-behavior: none` (מניעת bounce)
- Touch targets מינימום 44px
- `-webkit-tap-highlight-color: transparent`
- `vibrate(20ms)` על כל מהלך, `vibrate(80ms)` על ניצחון

### פתרון אוטומטי (💡 פתור)
- בלחיצה: קוביות זזות בזו אחר זו ופותרות את הפאזל לבד
- אלגוריתם: הערבוב מתועד ב-`picks[]`, הפתרון = `picks.slice(0,-1).reverse().concat([TOTAL-1])`
- מהירות קוביות מותאמת לאורך הפתרון (50–180ms בין מהלכים)

### סטטיסטיקות ומסך ניצחון
- שעון ומונה מהלכים
- שיאים אישיים נשמרים ב-`localStorage` לפי גודל לוח
- מסך ניצחון: תצוגת התמונה, זמן + מהלכים, השוואה לשיא, באנר 20% הנחה עם countdown 24 שעות
- כפתורים: לגלריה + הנחה / שתף בוואטסאפ / העתק קישור / פאזל חדש

### לינק הנחה
הכפתור "לתמונה בגלריה" מוביל אל:
```
https://amitphotos.com/?photo=<id>&discount=puzzle
```

## 2. אינטגרציית הנחת פאזל בגלריה (gallery.js)

### מה נוסף
- משתנה גלובלי `puzzleDiscountPhotoId` — מוגדר כשמגיעים עם `?discount=puzzle`
- קבוע `PUZZLE_DISCOUNT = 0.20`

### `handleInitialHash()`
כשמגיעים עם `?discount=puzzle&photo=ID`:
1. שומר `puzzleDiscountPhotoId = photoId`
2. פותח לייטבוקס לאותה תמונה
3. מציג בנר `puzzle-discount-banner` עם לינק וואטסאפ מוכן

### `getEffectivePrice()`
עדכון לוגיקת ההנחה — נבדק בסדר הבא:
1. `price_overrides` — הנחה ידנית ספציפית
2. `is_week_photo + week_photo_discount` — הנחת תמונת השבוע
3. `puzzleDiscountPhotoId` — הנחת פאזל 20%
4. מחיר רגיל

### `openBuyModal()`
- כשיש הנחת פאזל — מחיר עם קו (`<s>`) ומחיר מוזל לצד
- בדג': `🧩 הנחת פאזל` (במקום `🏷 מבצע`)

## 4. עדכוני סגנון כיתובים — סושיאל מדיה

### שינוי מדיניות כיתובים
**לפני:** סגנון שירי/רגשי בגוף ראשון ("ניצחתי את האור", "הרגע הזה הקפיא אותי")
**אחרי:** עובדתי + טכני — מה בפריים, ההחלטה הטכנית/קומפוזיציונית, נתוני EXIF

### מבנה כיתוב — instagram_post.py / facebook_post.py / week_photo_social.py
```
1. משפט: מה בפריים — נושא, ז'אנר (מאקרו/לנדסקייפ/פורטרט...), מיקום אם ניתן לזהות
2. משפט: ההחלטה הטכנית/קומפוזיציונית — תאורה, שעת יום, בחירת פריים
3. שורת EXIF (אם קיים): f/2.8 · 1/500s · 85mm · ISO 400 · Nikon Z6
4. שורה ריקה
5. לינק רכישה / "link in bio"
```

### שדות EXIF שנוספו (היו חסרים)
- `aperture`, `shutter`, `focal`, `iso`, `camera` — נאספים ומוצגים כשורת `exif_line`
- גם `facebook_post.py` קיבל `exif_parts` (לא היה לפני)

## 5. תיקוני GitHub Actions

### בעיה 1 — Anthropic 5MB limit
**שגיאה:** `image exceeds 5 MB maximum: 5544520 bytes > 5242880 bytes`

**סיבה:** `max_bytes=4_500_000` בדק raw bytes, אבל base64 מנפח ב-~33% → 4.5MB raw = ~6MB base64.

**תיקון** (בכל 3 סקריפטים: instagram, facebook, week_photo_social):
- `max_bytes=3_750_000` (3.75MB raw → ~5MB base64, מתחת ל-limit)
- `max_dim=2000` (הקטנת מימדים לפני הקידוד, במקום 7000)

### בעיה 2 — Instagram דוחה catbox.moe URLs
**שגיאה:** `Only photo or video can be accepted as media type. The media URI does not meet our requirements.`

**תיקון ב-instagram_post.py:**
1. מעדיף `photo.url` (URL מלא) על `photo.thumbnail`
2. אם ה-URL כבר ב-`amitphotos.com/` — משתמש ישירות (Cloudflare R2, מאושר על ידי Instagram)
3. אם לא — מעלה ל-R2 שלנו דרך `/api/admin/upload-story` (fallback: 0x0.st ואחרים)
4. `ADMIN_TOKEN` נוסף ל-instagram-post.yml

### תיקון נוסף
- `pillow` נוסף ל-`facebook-post.yml` (נדרש לדחיסת תמונות)
