# Newsletter — Design Spec
**Date:** 2026-05-19
**Status:** Approved

## Overview

A bi-monthly newsletter system for amitphotos.com. Content is pulled automatically from the site (photos, guides, locations), Hebrew copy is generated by Claude API, and the result is published as a magazine-style web page at `/newsletter/[slug]/`. The admin reviews and publishes via `/admin/newsletter/`. Phase B adds Mailchimp sending.

## Goals

- Surface new content (recently uploaded photos, new guides) + resurface existing content with a different angle each issue
- Magazine-quality Hebrew text, auto-generated and human-reviewed
- Web version lives on amitphotos.com (dark theme, Heebo/Syne, gold accent)
- PDF via browser print (no external libraries)
- Phase B: Mailchimp campaign sent to subscribers

## Issue Cadence

| Date | Type | Contents |
|---|---|---|
| 1st of month | `full` | Hero photo + guide spotlight + location + tip + fixed links |
| 15th of month | `flash` | One photo + one tip + sale reminder |

## Database Schema

New table added to D1 `amit-photos-db`:

```sql
CREATE TABLE IF NOT EXISTS newsletter_issues (
  id           TEXT PRIMARY KEY,
  slug         TEXT UNIQUE NOT NULL,
  type         TEXT NOT NULL DEFAULT 'full',
  issue_number INTEGER NOT NULL,
  title_he     TEXT NOT NULL DEFAULT '',
  title_en     TEXT NOT NULL DEFAULT '',
  content_json TEXT NOT NULL DEFAULT '{}',
  status       TEXT NOT NULL DEFAULT 'draft',
  published_at TEXT,
  created_at   TEXT NOT NULL DEFAULT (datetime('now'))
);
```

`content_json` shape for `flash` issues:

KV key `newsletter_rotation` stores rotation state:
```json
{ "last_guide_index": 4, "last_location_index": 2, "last_issue_number": 7 }
```

## Content Generation Pipeline

Runs on Cron (1st and 15th of month, existing `0 */2 * * *` schedule extended):

1. Read `newsletter_rotation` from KV
2. Check D1: if a draft/published issue already exists for this month+type, skip
3. Pick hero photo: most recently uploaded photo (`ORDER BY created_at DESC LIMIT 1`) with photo_id not matching the previous issue's hero photo_id
4. Pick guide: `(last_guide_index + 1) % 19` from ordered list of guide slugs
5. Pick location: `(last_location_index + 1) % total_locations` from D1 locations table
6. Call Claude API (`claude-opus-4-7`) with metadata for all sections → receive JSON with Hebrew + English text for each section
7. INSERT into `newsletter_issues` with `status = 'draft'`
8. Update `newsletter_rotation` in KV

Claude prompt structure:
- System: "אתה עורך ניוזלטר צילום. כתוב עברית תקנית וברורה. טון חם, לא פורמלי, לצלמים מתחילים ומתקדמים."
- User: photo title + category + description + guide title + location name → return JSON with `text_he` and `text_en` per section

## Worker Routes

### Public

```
GET /newsletter/
  → HTML: list of published issues (title, date, issue_number)
  → Shows most recent 12 issues

GET /newsletter/:slug/
  → HTML: full magazine-style issue page
  → 404 if status !== 'published'
```

### Admin (requires X-Admin-Password header)

```
GET  /admin/newsletter/
  → HTML: list of all issues (draft + published) with status badges

GET  /admin/newsletter/:id/
  → HTML: editor page — edit title, swap photo/guide/location, edit text fields
  → "Preview" button opens `/admin/newsletter/:id/preview/` — magazine layout rendered server-side, no status check (admin-auth route only)

POST /api/admin/newsletter/generate
  → Body: { type: "full" | "flash" }
  → Runs content pipeline immediately, returns { id, slug }

PATCH /api/admin/newsletter/:id
  → Body: partial content_json fields + title_he + title_en
  → Updates draft

POST /api/admin/newsletter/:id/publish
  → Sets status = 'published', published_at = now()
  → Returns { url: "/newsletter/:slug/" }
```

## Web Version Design

### `/newsletter/` (list page)

Dark theme, matches site. Grid of issue cards:
- Issue number + type badge (מלא / הבזק)
- Title
- Date published
- Hero photo thumbnail
- Link to full issue

### `/newsletter/:slug/` (magazine issue)

Full dark-theme magazine layout:

**Header:**
```
[Logo / Amit Photos]    גיליון #N | יוני 2026
─────────────────────────────────────────────
```

**Full issue sections:**
1. **Hero** — full-width photo (max 800px), title in Syne gold, 1-2 paragraph Hebrew text, "לניתוח המלא ←" link if photo has analysis
2. **הדרכה החודשית** — photo of guide hero (if exists) or icon, title, 2-sentence description, "קרא את המדריך ←"
3. **מקום לצילום** — location hero photo, name, when-to-visit, 2 sentences, "לכל המקומות ←"
4. **טיפ החודש** — gold card, short tip paragraph
5. **קישורים** — horizontal row of 4 site section links

**Flash issue sections:**
1. **תמונת השבועיים** — large photo + 2 sentences
2. **טיפ** — gold card
3. **מבצע** — photo thumbnail + caption

**Footer:**
```
© Amit Photos | להסרה מהרשימה: email
```

### Print CSS

`@media print` stylesheet:
- Background: white, text: black
- Page size: A4
- Photos retain borders, scale to page width
- Header repeated via `position: running(header)` or `<thead>` trick
- `page-break-inside: avoid` on each section card
- Hide: nav bar, unsubscribe footer link

## Admin Editor UI

`/admin/newsletter/:id/`:

- Title field (Hebrew)
- For each section: thumbnail preview + "החלף" button (opens photo picker from D1), textarea for Hebrew text, textarea for English text
- Section order matches final layout
- "שמור טיוטה" — PATCH to API
- "תצוגה מקדימה" — opens `/newsletter/:slug/?preview=1` (Worker ignores status for preview param + admin password cookie)
- "פרסם" — POST to publish endpoint, redirects to live URL

## Phase B — Mailchimp (future)

After Phase A ships:
- Add "שלח ב-Mailchimp" button in admin editor
- Generates simplified email HTML (table-based layout, inline CSS, no dark background for email client compat)
- Creates Mailchimp campaign via API with subscriber list
- Sends to all subscribers in D1 `subscribers` table synced to Mailchimp audience

Email differs from web: lighter background (dark gray `#1a1a1a` for Gmail compat), no CSS Grid, images hosted at amitphotos.com R2 URLs.

## i18n

- All rendered text uses `data-he` / `data-en` + `applyLang()` pattern identical to camera guide pages
- `getLang()` reads from localStorage
- `window.setLang = applyLang` exported for nav.js compatibility

## Technical Constraints

- No new npm packages — vanilla JS + Cloudflare Workers native APIs
- Claude API key: Worker secret `ANTHROPIC_API_KEY` (add via `wrangler secret put ANTHROPIC_API_KEY` if not present)
- KV namespace: use existing `PHOTOS_KV` binding for `newsletter_rotation` key
- Admin auth: existing `X-Admin-Password` header pattern (same secret as rest of admin)
- R2 photo URLs: `https://amitphotos.com/photos/<id>.jpg`
- Guide slugs (19 total): lenses, light, exposure, depth-of-field, filters, composition, white-balance, histogram, dynamic-range, editing, software, sports, macro, types, visual-language, controls, landscape, portrait, focus
- Cron schedule: add `0 9 1,15 * *` to wrangler.toml (9:00 on the 1st and 15th of each month)
