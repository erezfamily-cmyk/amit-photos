# Locations Expansion — Design Spec
תאריך: 2026-05-14

## סקירה

הרחבת פיצ'ר "מקומות לצילום" שכבר קיים באתר:
- **שלב 1:** שדרוג 6 דפי מקום קיימים עם 3 סעיפים חדשים
- **שלב 2:** הוספת 7 דפי מקום חדשים (4 חו"ל + 3 ישראל)

אין שינויים ב-D1 schema, ב-Worker routes, או ב-hub page — רק הרחבת תוכן ועיצוב דף הspot.

---

## שלב 1 — שדרוג דף המקום הבודד (`/locations/spot/index.html`)

### 3 סעיפים חדשים

#### א. "מתי לבוא" (`#when-to-visit`)
- רשת 4 עונות (קיץ / סתיו / חורף / אביב) עם דירוג: ⭐ מצוין / ✅ טוב / ⚠️ פחות מומלץ
- כל עונה כוללת הסבר קצר (מזג אוויר, עומס תיירים, איכות האור)
- ה-AI מייצר את הנתונים בעת יצירת/עשרוי המקום (enrichment)
- שדה ב-D1: `when_to_visit` — JSON string, מבנה: `{"summer":{"rating":"bad","note":"חם מאוד"},"fall":{"rating":"ok","note":"..."},"winter":{"rating":"good","note":"..."},"spring":{"rating":"ok","note":"..."}}`
  - rating אפשרי: `"good"` (⭐ מצוין) / `"ok"` (✅ טוב) / `"bad"` (⚠️ פחות מומלץ)

#### ב. "ציוד מומלץ" (`#gear`)
- שבבים (chips) עם עדשות ואביזרים
- עדשה עיקרית מסומנת בצבע זהב, שאר — אפורות
- ה-AI מייצר per-location לפי סוג הצילום (נוף, עירוני, חיות וכו')
- שדה ב-D1: `recommended_gear` — JSON array של strings

#### ג. "מקומות קרובים" (`#nearby`)
- רשימה של עד 3 מקומות אחרים, ממוינים לפי מרחק (Haversine מ-coordinates)
- מחושב ב-Worker בזמן GET /api/locations/:slug — לא שדה ב-D1
- כרטיסייה עם שם + מרחק בק"מ + קישור לדף

### שינויים טכניים לשלב 1

**Worker (`worker.js`):**
- GET /api/locations/:slug — מוסיף `nearby` לתשובה: שולף את כל המקומות, מחשב מרחק Haversine, מחזיר 3 הקרובים ביותר (לא כולל המקום עצמו)
- POST /api/admin/locations ו-PUT /api/admin/locations/:slug — prompt ה-AI מורחב לייצר גם `when_to_visit` + `recommended_gear`

**D1 (`locations` table):**
- `ALTER TABLE locations ADD COLUMN when_to_visit TEXT` — JSON string
- `ALTER TABLE locations ADD COLUMN recommended_gear TEXT` — JSON string

**Frontend (`/locations/spot/index.html`):**
- מוסיף 3 סעיפים לאחר "טיפים לצילום" ולפני "גלריה"
- מקומות קרובים — לאחר הגלריה, לפני "מדריכים קשורים"

**Admin (`admin.html`):**
- מוסיף 2 שדות עריכה בטאב מקומות: `when_to_visit` (textarea JSON) + `recommended_gear` (textarea JSON)
- כפתור "העשר מחדש" קיים כבר — יעדכן גם את השדות החדשים

### עיצוב

```
[עונות — 4 columns]
 קיץ        סתיו       חורף       אביב
⚠️ חם      ✅ טוב    ⭐ מצוין   ✅ טוב
 טקסט       טקסט       טקסט      טקסט

[ציוד מומלץ — chips]
 [עדשה רחבה 16-35mm]  [מסנן ND]  [חצובה]  [מקטב]
  (זהב, ראשי)

[מקומות קרובים — list]
 מצדה                           22 ק"מ →
 עין גדי                        35 ק"מ →
```

## שלב 2 — 7 דפי מקום חדשים

### סדר יצירה + מקור תמונות

| # | מקום | אזור | תמונות בגלריה | הערות |
|---|------|------|---------------|-------|
| 1 | מסגד שייח זאיד, אבו דאבי | חו"ל | 53 (קטגוריה "אבו דאבי") | מספיק תמונות |
| 2 | הדולומיטים, איטליה | חו"ל | 58 (קטגוריה "איטליה") | פסגות, אגמים, שלג |
| 3 | ספארי, טנזניה | חו"ל | 28 (קטגוריה "טנזניה") | חיות + נוף |
| 4 | מנזרי מטאורה, יוון | חו"ל | 38 (קטגוריה "יוון") | כולל מפלות |
| 5 | כנרת | ישראל — צפון | 2-3 | ייתכן שיצריך הורדה מ-Drive |
| 6 | הר חרמון בשלג | ישראל — צפון | 2-3 | עונתי, ינואר–מרץ |
| 7 | גנים בהאי, חיפה | ישראל — צפון | 2 | ייתכן שיצריך הורדה מ-Drive |

### תהליך יצירת כל דף
1. POST /api/admin/locations — AI מייצר enrichment (כולל when_to_visit + recommended_gear)
2. POST /api/admin/locations/:slug/photos — קישור תמונות מהגלריה (gallery type)
3. בדיקה ידנית באדמין — תיקון אם צריך

### אסטרטגיית תמונות
- **חו"ל:** תמונות "gallery" מהקטגוריות הקיימות — ללא הורדות
- **ישראל (כנרת, חרמון, חיפה):** אם אין מספיק בגלריה — סקריפט Python שמשתמש ב-token.json הקיים לשליפה מ-Drive ישירות

---

## מה לא כלול בתכנית

- שינוי ב-hub page (`/locations/index.html`) — לא נדרש
- "מסלול צילום" / route pages — גישה B, לא כלולה עכשיו
- GPX download — אופציונלי, לא בשלב זה
- רמת קושי — אופציונלי, לא בשלב זה
- Google Photos API (scope נפרד) — לא נדרש, Drive מספיק

## סדר ביצוע

```
1. ALTER TABLE (2 עמודות חדשות ב-D1)
2. עדכון AI prompt ב-Worker (enrichment)  
3. עדכון GET /api/locations/:slug (חישוב nearby)
4. עדכון /locations/spot/index.html (3 סעיפים)
5. עדכון admin.html (2 שדות)
6. Deploy Worker + בדיקה
7. העשרה מחדש של 6 הדפים הקיימים (כפתור "העשר" באדמין)
8. יצירת 7 דפים חדשים + שיוך תמונות
```
