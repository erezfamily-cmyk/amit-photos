# משחק קוויז "מאיפה הצילום?" — Design Spec
**תאריך:** 2026-04-30

## סקירה
דף משחק עצמאי (`quiz/index.html`) — קוויז של 5 שאלות "מאיפה הצילום?". מציג תמונה של אתר מוכר מהגלריה של עמית, המשתמש מנחש מתוך 4 מדינות. 3+ נכון → הנחה 20%. מסך ניצחון עם שיתוף וקישור לגלריה.

---

## ארכיטקטורה

### קבצים חדשים
- `quiz/index.html` — דף המשחק (HTML + CSS + JS בקובץ אחד, ללא תלויות חיצוניות)
- `src/scan_quiz_photos.py` — סקריפט חד-פעמי: Claude Vision סורק תמונות מקטגוריות נסיעות ומסמן quiz_eligible + quiz_description
- `.github/workflows/quiz-social-post.yml` — פרומו שבועי לסושיאל

### שינויים בקבצים קיימים
- `worker.js` — שדות `quiz_eligible` ו-`quiz_description` ב-API + endpoint לעדכון
- `assets/js/gallery.js` — זיהוי `?discount=quiz`, הפעלת הנחה
- `admin.html` — כפתור 🌍, בדג' `🌍 קוויז`, טוגל quiz_eligible

## תמונות — בחירה וסינון

### קטגוריות כשירות (יש מקומות מזוהים)
ישראל, איטליה, אנגליה, יוון, וינה, הולנד, גרמניה, טנזניה, מונטנגרו, סלובקיה, סן דיאגו, ספרד ואנדורה, צכיה, אבו דאבי

### תהליך הסימון (scan_quiz_photos.py)
1. מושך כל תמונות מהקטגוריות הכשירות מה-API
2. לכל תמונה — שולח ל-Claude Vision עם prompt:
   ```
   Does this photo show a recognizable landmark, building, or location that people would clearly identify?
   If yes: return {"eligible": true, "description": "one sentence in Hebrew: Name — brief factual description, City, Country"}
   If no (nature, flowers, people, abstract): return {"eligible": false}
   ```
3. שומר לכל תמונה: `quiz_eligible: true/false` + `quiz_description: "..."` דרך PATCH /api/photos

### עדכון ידני מאדמין
- כפתור 🌍 על כל תמונה — טוגל `quiz_eligible` (מופעל/כבוי)
- כשמופעל → Claude מייצר `quiz_description` אוטומטית אם חסר

## לוגיקת המשחק

### טעינה
- GET `/api/photos` → מסנן רק `quiz_eligible: true`
- בוחר 5 תמונות רנדומליות

### מבנה שאלה
- מציג תמונה (thumbnail)
- 4 כפתורי מדינה: הנכונה (מהקטגוריה) + 3 אחרות רנדומליות מהpool
- תמיד 4 אפשרויות שונות

### Pool מדינות לתשובות שגויות
```javascript
const COUNTRIES = [
  "🇮🇱 ישראל", "🇮🇹 איטליה", "🇬🇧 אנגליה", "🇬🇷 יוון",
  "🇦🇹 וינה", "🇳🇱 הולנד", "🇩🇪 גרמניה", "🇹🇿 טנזניה",
  "🇲🇪 מונטנגרו", "🇸🇰 סלובקיה", "🇺🇸 סן דיאגו", "🇪🇸 ספרד",
  "🇨🇿 צ'כיה", "🇦🇪 אבו דאבי"
];
```

### פידבק מיידי
אחרי ניחוש:
- ✅ נכון — כפתור הנכון מתאדם ירוק
- ❌ טעות — כפתור שלחצו אדום, הנכון מתאדם ירוק
- מופיע תיאור המקום: `quiz_description` (שורה אפורה מתחת לתמונה)
- כפתור "הבא" לשאלה הבאה

### פס התקדמות
`שאלה 3 / 5` + דוט-ניקוד (✓✓✗) + `✓ 2 נכון`

### ניצחון
- 3+/5 נכון → מסך ניצחון + הנחה 20%
- 0-2 נכון → מסך "נסה שוב" ללא הנחה + כפתור סיבוב חדש

---

## מסך ניצחון

תוכן:
1. תמונת השאלה האחרונה (thumbnail)
2. בדג' "🌍 סיבוב הושלם!"
3. 🏆 כל הכבוד!
4. "ענית נכון על X מתוך 5 שאלות"
5. 5 דוטים ✓/✗
6. באנר: "🎁 זכית בהנחה 20% על כל תמונה בגלריה — בתוקף 24 שעות"
7. כפתורים:
   - "🛍️ לגלריה עם ההנחה" → `https://amitphotos.com/?discount=quiz`
   - "💬 שתף בוואטסאפ"
   - "🔄 סיבוב חדש"

## אינטגרציה עם הגלריה (gallery.js)

כשמגיעים עם `?discount=quiz`:
- `puzzleDiscountPhotoId` מוחלף ב-`activeDiscount` עם סוג (`puzzle` / `quiz`)
- `getEffectivePrice()` מחיל 20% הנחה על **כל** התמונות (לא רק אחת)
- בדג': `🌍 הנחת קוויז`
- הבאנר בלייטבוקס מתעדכן בהתאם

> **שימו לב:** הפאזל נותן הנחה לתמונה ספציפית. הקוויז — לכל הגלריה (כי המשתמש ניחש 5 תמונות שונות).

## אדמין (admin.html)

### בדג' חדש
`🌍 קוויז` — מוצג על כרטיסיית תמונה כשיש `quiz_eligible: true`
(צבע כחול-ירוק, דומה לבדג' ⭐ שבוע)

### כפתור 🌍
בשורת הפעולות של כל תמונה — לחיצה מעתיקה ללוח:
```
https://amitphotos.com/?discount=quiz
```
(לינק כללי, לא לתמונה ספציפית)

### טוגל quiz_eligible
לחיצה ממושכת / לחיצה כפולה על כפתור 🌍 → פותחת popup קטן:
- "סמן כמתאים לקוויז" / "הסר מהקוויז"
- אם מסמנים ואין quiz_description → Claude מייצר אוטומטית

## פרומו שבועי (quiz-social-post.yml)

אותו pattern כמו `puzzle_social_post.py`:
- רץ כל יום רביעי 18:00 ישראל (`0 15 * * 3`)
- בוחר תמונה רנדומלית עם `quiz_eligible: true`
- Claude מייצר כיתוב: "מה המקום הזה?" + הזמנה לקוויז + "20% הנחה למנצחים"
- מפרסם לאינסטגרם + פייסבוק
